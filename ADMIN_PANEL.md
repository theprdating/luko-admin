## 1. 技術選型

### 選定：純 HTML + Supabase JavaScript SDK

```
❌ Flutter Web — 打包大、載入慢、審核體驗差
❌ Next.js / React — 架設需要時間，MVP 不必要的複雜度
✅ 純 HTML + Vanilla JS + Supabase JS SDK

優點：
  - 不需要建置流程（無需 npm build）
  - 直接放到任何靜態網站托管（Netlify / GitHub Pages 免費）
  - 開發時間：2–3 天
  - 一個 index.html 就能跑

部署：
  - Netlify（免費方案即可）
  - 或直接用 Supabase Edge Functions 托管（更整合）
```

## 2. 功能規格

### 必要功能（MVP）

| # | 功能 | 優先度 |
|---|------|--------|
| 1 | 管理員登入（Email + 密碼） | ✅ 必做 |
| 2 | 待審核申請列表 | ✅ 必做 |
| 3 | 查看申請者照片（大圖） | ✅ 必做 |
| 4 | 通過 / 拒絕按鈕 | ✅ 必做 |
| 5 | 審核備註（內部，不對外顯示） | ✅ 必做 |
| 6 | 已審核記錄查詢 | ✅ 必做 |
| 7 | 儀表板數字（待審 / 今日審核數） | ⏳ 後補 |
| 8 | 檢舉管理 | ⏳ 後補 |

### 不做的功能（初期）

```
❌ 用戶管理（直接用 Supabase Dashboard）
❌ 統計圖表（數據量太小，初期無意義）
❌ 多管理員角色權限（只有 co-founder，不需要）
❌ 行動裝置優化（審核最好在桌機上進行）
```

### 3-4. 審核動作確認

```
點「通過」：
  ─ 直接執行（不需彈窗，速度優先）
  ─ 執行：
      UPDATE applications SET status='approved', reviewed_at=NOW(), reviewed_by=...
      INSERT INTO profiles (從 applications 資料建立)
      觸發 Edge Function 發通知

點「拒絕」：
  ─ 直接執行（不需彈窗）
  ─ 執行：
      UPDATE applications SET status='rejected', reviewed_at=NOW(),
        reviewed_by=..., review_note=..., reapply_after=NOW()+INTERVAL '30 days'
      觸發 Edge Function 發通知

審核後行為：
  ─ 該筆申請從「待審核列表」消失
  ─ 顯示短暫成功提示（Toast）
  ─ 自動載入下一筆
```

## 4. 安全設計

### 管理員帳號設定

```
管理員帳號使用 Supabase Auth 的 User Metadata 標記 role：

在 Supabase Dashboard → Authentication → Users：
  找到管理員 user → Edit → Raw user meta data：
  { "user_role": "admin" }

程式碼中驗證：
  const { data: { user } } = await supabase.auth.getUser()
  if (user.user_metadata.user_role !== 'admin') {
    // 跳轉回登入頁
  }
```

### 安全規則

```
✅ 所有 API 呼叫都需要 Supabase session（JWT 驗證）
✅ RLS Policy 已限制只有 admin role 能操作 applications
✅ 網頁不對外公開（無入口連結），URL 不索引
✅ 每次進入頁面都重新驗證 session
✅ 照片使用 Signed URL（1 小時過期），不直連 Storage
❌ 不在後台網頁儲存任何密碼或 API Key（全走 Supabase SDK）
```

## 5. 建置方式

### 檔案結構

```
D:/Startup/Luko/admin/
├── index.html        ← 登入頁
├── dashboard.html    ← 主儀表板
├── review.html       ← 審核列表
├── history.html      ← 已審核記錄
├── style.css         ← 共用樣式（極簡，不需框架）
└── supabase.js       ← Supabase 初始化與共用函數
```

### 開發時程估算

```
Day 1：登入頁 + Supabase 連線 + 驗證機制
Day 2：待審核列表 + 照片顯示 + 通過/拒絕功能
Day 3：已審核記錄 + UI 細節調整 + 測試
```

### 部署（Netlify）

```bash
# 只需要把 admin/ 資料夾拖拉到 Netlify 即可

# 記得在 Netlify 設定環境變數：
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# 或直接在 supabase.js 中寫死（因為 anon key 本來就是公開的）
```
